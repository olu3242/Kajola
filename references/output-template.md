```markdown
### 2.2 Service Inventory

| Service | Provider | Purpose | Auth Method |
|---|---|---|---|
| Authentication | Supabase Auth | Phone OTP, session management | JWT |
| Database | Supabase Postgres 15 | Primary data store | Service role / RLS |
| File Storage | Supabase Storage | Profile photos, documents | Signed URLs |
| Realtime | Supabase Realtime | Live booking status, chat | JWT |
| API Logic | Supabase Edge Functions | Business logic, webhooks | Bearer / Service role |
| Primary Payment | Paystack | NGN payments, split, refunds | Secret key (server-only) |
| Pan-Africa Payment | Flutterwave | Multi-currency payments | Secret key (server-only) |
| East Africa Payment | M-Pesa Daraja | KES STK Push | OAuth 2.0 |
| SMS | Termii | OTP delivery, transactional SMS | API key (server-only) |
| WhatsApp | Twilio | OTP fallback, rich notifications | Account SID + Auth token |
| Maps | Google Maps Platform | Location search, distance calc | API key (restricted) |
| Push Notifications | Expo Push Service | Mobile push alerts | Expo access token |
| CDN | Cloudflare | Static asset delivery, DDoS | Zone API key |

### 2.3 Data Flow Narratives

#### Flow 1: User Registration (Phone OTP)
1. User enters phone number in app
2. App calls Edge Function `POST /auth/send-otp`
3. Edge Function: rate-check (max 3/10min), generate 6-digit OTP, bcrypt hash, insert `phone_otps` row, call Termii `send-otp` API
4. Termii delivers SMS to user's phone
5. User enters OTP in app
6. App calls Edge Function `POST /auth/verify-otp`
7. Edge Function: find unexpired row, compare bcrypt hash, mark verified, call `supabase.auth.admin.createUser` or sign in
8. Supabase returns JWT — app stores in SecureStore (mobile) or httpOnly cookie (web)
9. Edge Function creates `users` + `profiles` row for new users
10. App navigates to onboarding flow

#### Flow 2: [Core Domain Transaction — e.g. Booking Creation]
[Write 8–12 steps covering the happy path end-to-end]

#### Flow 3: Payment Capture
1. Client initiates payment — app calls Edge Function `POST /payments/initiate`
2. Edge Function creates `transactions` row (status: `pending`), calls Paystack `initialize` API
3. Paystack returns authorization URL / reference
4. App opens Paystack checkout (WebView or redirect)
5. User completes payment on Paystack
6. Paystack sends webhook to Edge Function `POST /webhooks/paystack`
7. Edge Function: verify HMAC signature, check idempotency key, update `transactions` row (status: `success`)
8. Edge Function triggers automation event `payment.confirmed`
9. Automation job: credit provider wallet, update booking status, send SMS confirmation
10. Realtime broadcasts booking status change to both client and provider apps

#### Flow 4: Notification Dispatch
1. Automation event fires (e.g. `booking.confirmed`)
2. Notification job reads user preferences and contact info
3. For each channel (SMS, push, WhatsApp): check user opt-in, format message, call provider API
4. Log delivery result to `notification_logs` table
5. On failure: mark failed, retry up to 3 times with exponential backoff
```

## Section 4 — API Template

```markdown
## Section 4 — API Definitions

Base URL: `https://[project-ref].supabase.co/functions/v1`
All authenticated endpoints require: `Authorization: Bearer <jwt>`

### Auth

---
POST /auth/send-otp
Auth: none
Description: Send a 6-digit OTP to the given phone number via SMS.

Request:
{
  "phone": string   // E.164 format, e.g. "+2348012345678"
}

Response 200:
{
  "message": "OTP sent",
  "expires_in": 600   // seconds
}

Errors:
- 400: Invalid phone format
- 429: Rate limit exceeded (max 3 OTPs per phone per 10 minutes)
---

POST /auth/verify-otp
Auth: none
Description: Verify OTP and return a Supabase session.

Request:
{
  "phone": string,
  "otp": string     // 6-digit code
}

Response 200:
{
  "access_token": string,
  "refresh_token": string,
  "user": { "id": uuid, "phone": string, "role": string }
}

Errors:
- 400: Missing fields
- 401: Invalid or expired OTP
- 429: Too many verification attempts (max 5 per OTP)
---
```

## Sections 6–11 — Abbreviated Templates

For sections 6–11, the template keys are:

**Section 6 — Monorepo**: Full directory tree with every package named and described. List all Edge Functions with their HTTP method, path, and purpose.

**Section 7 — Automation**: Complete event catalogue table (minimum 8 events). Full `automation_jobs` schema. All cron jobs with exact pg_cron schedules. HMAC verification pseudocode for each payment provider.

**Section 8 — Deployment**: Numbered steps (no vague instructions). Every env var in table format. GitHub Actions YAML outline (not pseudocode — actual job names and steps). Staging vs production config table.

**Section 9 — Monetization**: All rates as actual numbers (never "X%" or "TBD"). Revenue projections at 1k/10k/100k transactions. Paystack split payment configuration steps.

**Section 10 — Scaling**: Concrete actions at each threshold. Specific Supabase settings to change. Index strategies tied to query patterns. Cost estimates where relevant.

**Section 11 — Roadmap**: Weekly milestones for Q1. Feature-level milestones for Q2–Q4. Business impact for every milestone (not just technical deliverables).
